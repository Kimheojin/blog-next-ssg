## 📝 포스트 작성 가이드 (템플릿)

새로운 글을 작성할 때는 `public/posts/[카테고리명]/파일명.md` 경로에 아래 형식으로 작성하세요.

```markdown
---
title: "포스트 제목을 여기에 입력하세요"
date: "2026-02-26"
---

# 여기에 본문 제목 (h1)

여기에 내용을 작성하시면 됩니다. 
마크다운 문법을 그대로 지원합니다.

- 리스트 아이템 1
- 리스트 아이템 2

이미지 사용 시:
![설명](/images/posts/이미지파일명.png)
```

**주의사항:**
1. 상단의 `---` 사이(Frontmatter)에는 `title`과 `date`가 반드시 포함되어야 합니다.
2. `date` 형식은 `YYYY-MM-DD`를 권장합니다.
3. 카테고리는 파일이 위치한 **폴더 이름**에 따라 자동으로 결정됩니다.

## 🚀 배포 가이드 (GitHub Pages)

이 프로젝트는 `out` 디렉토리의 정적 파일들을 별도의 배포 전용 저장소(`HeoJin1109.github.io`)로 푸시하여 배포하도록 설정되어 있습니다.

### 배포 전 필수 확인
- `public/.nojekyll` 파일이 존재해야 합니다. (GitHub Pages의 Jekyll 무시 설정 방지)
- `next.config.ts`의 `output: "export"`와 `trailingSlash: true` 설정이 되어 있어야 합니다.

### 배포 명령어
아래 명령어를 통해 빌드와 배포를 한 번에 진행할 수 있습니다.

```bash
# 1. 최신 코드로 정적 파일 생성 (out 폴더 생성)
npm run build

# 2. out 폴더의 내용을 지정된 배포용 repo의 main 브랜치로 푸시
npm run deploy
```

> **참고:** `npm run deploy` 실행 시 GitHub 인증을 요구할 수 있습니다. 이 명령어는 `gh-pages` 패키지를 사용하여 `out` 폴더 내의 파일들만 배포용 저장소의 `main` 브랜치로 깔끔하게 전송합니다.

### ⚠️ 배포 시 주의사항
- **`npm run build`**는 현재 소스 코드를 바탕으로 로컬 `out` 디렉토리에 정적 파일들을 생성하는 과정입니다. (실제 서버에 반영되지 않음)
- **`npm run deploy`**는 로컬에 생성된 `out` 디렉토리의 내용들을 실제 웹사이트가 호스팅되는 `HeoJin1109.github.io` 저장소의 `main` 브랜치로 **전송(Push)**하는 과정입니다.
- **결론:** 웹사이트에 최신 내용을 반영하려면 반드시 **`npm run build` 실행 후 `npm run deploy`를 이어서 실행**해야 합니다.

### 구글 서치 엔진 최적화 관련

  1. 기술적 SEO (검색 엔진 수집 최적화)
   * Sitemap (sitemap.xml) 생성: 검색 엔진이 블로그의 모든 포스트 경로를 한눈에 파악하도록 app/sitemap.ts를 통해 동적 사이트맵을 생성합니다.
   * robots.txt 설정: 검색 엔진 크롤러에게 수집 허용 범위와 사이트맵 위치를 명확히 알려줍니다.
   * Canonical URL 설정: 동일 콘텐츠가 여러 URL로 인식되는 것을 방지하기 위해 각 페이지의 대표 URL을 메타데이터에 명시합니다.


  2. 메타데이터 최적화 (검색 결과 노출 강화)
   * 동적 메타데이터 적용: 각 포스트의 마크다운 제목과 카테고리를 활용해 페이지마다 고유한 <title>과 <meta description>을 생성합니다.
   * Open Graph & Twitter Cards: 페이스북, 카카오톡, 트위터 등에 공유될 때 포스트 제목과 설명이 예쁘게 나오도록 설정합니다. 이는 간접적으로 클릭률(CTR)을
     높여 SEO에 도움을 줍니다.


  3. 구조화 데이터 (검색 결과 리치 결과 반영)
   * JSON-LD (Schema.org) 삽입: BlogPosting 스키마를 사용하여 검색 엔진이 이 페이지가 "블로그 게시물"임을 공식적으로 이해하게 합니다. 검색 결과에 작성자,
     발행일 등이 표시될 확률이 높아집니다.


  4. 콘텐츠 및 성능 최적화
   * 이미지 SEO: 마크다운 내 이미지에 alt 속성을 강제하거나, Next.js의 Image 컴포넌트를 연동하여 이미지 크기 최적화 및 지연 로딩을 적용합니다.
   * 시맨틱 태그 점검: 현재 이미 <article>, <h1>, <time> 등을 잘 쓰고 계시지만, 본문 내 헤딩(h2, h3) 구조가 논리적으로 이어지는지 확인합니다.
   * 파비콘 및 매니페스트: 사이트의 정체성을 나타내는 다양한 크기의 아이콘과 manifest.json을 추가합니다.

## 🖼️ 이미지 추가 및 관리 가이드

블로그 포스트에 이미지를 추가할 때는 아래와 같은 방식을 권장합니다.

*   **저장 위치**: `public/images/` 폴더 내부에 이미지 파일을 저장하세요. (예: `public/images/my-photo.webp`)
*   **마크다운 호출**: 마크다운 파일에서 루트 경로(`/`)를 기준으로 이미지를 불러옵니다.
    *   예시: `![이미지 설명](/images/my-photo.webp)`
*   **정적 배포 최적화**: 현재 `next.config.ts`에 `unoptimized: true` 설정이 되어 있어, GitHub Pages와 같은 정적 호스팅 환경에서도 이미지가 깨지지 않고 잘 로드됩니다.
*   **권장 사항**: 페이지 로딩 성능을 위해 `WebP` 또는 `AVIF` 형식을 사용하는 것이 좋습니다.
